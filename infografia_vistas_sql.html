¿Qué es una vista?
📋

Tabla virtual

No almacena datos físicamente en disco. Consulta las tablas base en tiempo real cada vez que es invocada por el sistema.

🔒

Seguridad y control

Restringe el acceso a columnas o filas específicas, ocultando información confidencial a usuarios sin privilegios.

🔄

Independencia lógica

Aísla cambios estructurales de las tablas. Si el esquema cambia, la vista se adapta sin romper las aplicaciones.

⚡

Simplificación

Encapsula JOINs y filtros complejos en una estructura simple y reutilizable para cualquier usuario.

-- Vista 4.7: Relación paciente ↔ enfermedad
CREATE VIEW v_pacienteEnfermedad (
  codigoP, nombresP, paternoP, maternoP,
  codigoD, nombreD, nombreE
) AS
SELECT
  p.codP, p.nombresP, p.paternoP, p.maternoP,
  d.codD, d.descripcionD, e.nombreE
FROM tpaciente p
JOIN tdiagnostico d
  ON p.codP = d.codP
JOIN tenfermedadeneldiagnostico ed
  ON d.codD = ed.codD
JOIN tenfermedad e
  ON ed.codE = e.codE;

SELECT * FROM v_pacienteEnfermedad;
-- Vista 5.1: Médicos y sus especialidades
CREATE VIEW v_MedicoEspecialidades AS
SELECT
  m.codM, m.nombresM, m.paternoM, m.maternoM,
  e.nombreEsp, em.fechaDeObtencionDeLaEspecialidadEspM
FROM tMedico m
JOIN tEspecialidadDelMedico em
  ON m.codM = em.codM
JOIN tEspecialidad e
  ON em.codEsp = e.codEsp;

-- 5.2: Cuántas especialidades tiene cada médico
SELECT nombresM, paternoM,
  COUNT(*) AS cantidad_especialidades
FROM v_MedicoEspecialidades
GROUP BY nombresM, paternoM;

-- 5.3: Solo médicos con 2 o más especialidades
SELECT nombresM, paternoM,
  COUNT(*) AS cantidad_especialidades
FROM v_MedicoEspecialidades
GROUP BY nombresM, paternoM
HAVING COUNT(*) >= 2;
-- Vista 5.4: Enfermedades aún no detectadas
CREATE VIEW v_EnfermedadesNoDetectadas AS
SELECT * FROM tEnfermedad
WHERE codE NOT IN (
  SELECT codE
  FROM tEnfermedadEnElDiagnostico
);

SELECT * FROM v_EnfermedadesNoDetectadas;

-- Vista 5.5: Especialidades sin médico asignado
CREATE VIEW v_EspecialidadesSinMedico AS
SELECT * FROM tEspecialidad
WHERE codEsp NOT IN (
  SELECT codEsp
  FROM tEspecialidadDelMedico
);

-- 5.6: Total de especialidades sin médico
SELECT COUNT(*) AS total_sin_medico
FROM v_EspecialidadesSinMedico;
-- Vista 5.7: Pacientes sin enfermedad en diagnóstico
CREATE VIEW v_PacientesSinEnfermedad AS
SELECT DISTINCT
  p.nombresP, p.paternoP, p.maternoP
FROM tPaciente p
JOIN tDiagnostico d
  ON p.codP = d.codP
WHERE d.codD NOT IN (
  SELECT codD
  FROM tEnfermedadEnElDiagnostico
);

SELECT * FROM v_PacientesSinEnfermedad;

Conclusiones
1
Las vistas simplifican el acceso a datos complejos, encapsulando JOINs múltiples en estructuras reutilizables que cualquier usuario puede consultar sin conocer el esquema interno.
2
El uso de subconsultas con NOT IN permite identificar registros huérfanos o sin relación, como enfermedades sin diagnóstico o especialidades sin médico asignado.
3
La combinación de vistas con GROUP BY y HAVING permite análisis estadísticos avanzados sobre datos distribuidos en múltiples tablas relacionales de forma eficiente.
